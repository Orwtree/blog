OrwtreeTeams

v0.2.1 2026-07-10 Java · Socket MIT License

OrwtreeTeams 是一款轻量级 局域网即时通讯 工具。 采用 Java 原生 Socket + 多线程技术实现，无需中心服务器，开箱即用。 适合 团队内部沟通、小型办公室、局域网协作 等场景。

✅ 当前版本支持群聊、文件传输、消息加密等核心功能，已在 Windows / macOS / Linux 上测试通过。

核心特性

  • 💬 即时通讯 — 支持群聊、私聊，消息实时推送，延迟 < 100ms
  • 📁 文件传输 — 支持拖拽发送文件，断点续传，传输进度可视
  • 🔐 消息加密 — 使用 AES-256 对称加密，保障通信内容安全
  • 👥 群组管理 — 支持创建/加入群组，成员列表实时同步
  • 📶 零配置 — 自动发现局域网内其他在线用户，无需手动设置 IP
  • 🎨 跨平台 — 基于 Java，支持 Windows、macOS、Linux

架构设计

OrwtreeTeams 采用 P2P 去中心化 架构，每个节点既是客户端也是服务端：

┌─────────────────────────────────────────────────────┐
│                   应用层 (UI)                        │
│         Java Swing / JavaFX (规划中)               │
├─────────────────────────────────────────────────────┤
│                   业务逻辑层                         │
│   消息路由 · 群组管理 · 文件传输 · 加密解密         │
├─────────────────────────────────────────────────────┤
│                   网络层                            │
│    Java Socket · 多线程 · UDP 发现 · TCP 通信      │
├─────────────────────────────────────────────────────┤
│                   传输协议                          │
│    自定义二进制协议 · 序列化 · 心跳保活            │
└─────────────────────────────────────────────────────┘
💡 设计目标：轻量、可靠、易扩展。核心代码量控制在 5000 行以内，便于二次开发。

快速开始

📦 下载安装

从 Releases 页面 下载对应平台的 JAR 包或安装包， 双击运行即可。需要 Java 11+ 运行环境。

🏗️ 从源码构建

# 克隆仓库
git clone https://github.com/OrwtreeStudio/OrwtreeTeams.git

# 使用 Maven 构建
mvn clean package

# 运行
java -jar target/orwtree-teams-0.2.1.jar

🚀 首次使用

  • 启动后，程序会自动扫描局域网内的其他 OrwtreeTeams 用户
  • 输入用户名，点击「登录」即可进入主界面
  • 在群组列表中创建新群组或加入已有群组
🎯 提示：确保防火墙允许 OrwtreeTeams 使用 TCP 8887~8890 和 UDP 8890 端口。

通信协议

📡 消息格式

所有消息采用 二进制协议，头部包含消息类型、长度、校验和等信息：

// 消息头 (固定 16 字节)
struct MessageHeader {
    uint8  magic;      // 魔数: 0x4F (即 'O')
    uint8  version;    // 协议版本
    uint16 type;       // 消息类型: 0x01 文本 | 0x02 文件 | 0x03 心跳
    uint32 length;     // 消息体长度
    uint32 checksum;   // CRC32 校验
    uint32 timestamp;  // 时间戳
};

🔐 加密流程

  • 首次连接时通过 Diffie-Hellman 密钥交换建立共享密钥
  • 后续通信使用 AES-256-GCM 加密，每条消息独立 IV
  • 密钥每 30 分钟自动轮换，保障长期通信安全

💓 心跳保活

  • 客户端每 30 秒 向已知节点发送心跳包
  • 连续 3 次 未响应则标记为离线
  • 支持离线消息缓存（最多 1000 条）
⚠️ 当前版本加密为 Database二进制，AES-256-GCM，默认不对外开放
